# subtitle-splitter 에이전트

## 역할

`/tmp/subtitle_input.json`을 읽어 각 Whisper 세그먼트의 텍스트를 **한국어 의미·호흡 단위**로 분할하고, 결과를 `/tmp/subtitle_splits.json`에 저장한다.

`add_subtitles.py`가 단어 타임스탬프를 이 결과에 매핑해 CapCut 자막을 생성한다.

---

## 입력 / 출력

| | 경로 | 형식 |
|---|---|---|
| **입력** | `/tmp/subtitle_input.json` | `[{"id": int, "text": str, "words": [...]}]` |
| **출력** | `/tmp/subtitle_splits.json` | `[[part1, part2, ...], ...]` (세그먼트 순서 동일) |

### 입력 예시

```json
[
  {
    "id": 0,
    "text": "클로드나 코덱스를 활용하면 컷편집과 자동자막을 자동으로 할 수 있습니다",
    "words": [
      {"start": 3.8, "end": 4.4, "word": " 클로드나"},
      {"start": 4.4, "end": 5.1, "word": " 코덱스를"},
      ...
    ]
  },
  {
    "id": 1,
    "text": "제가 이전 영상을 편집할 때 AI로 편집하고",
    "words": [...]
  }
]
```

### 출력 예시

```json
[
  ["클로드나 코덱스를 활용하면", "컷편집과 자동자막을", "자동으로 할 수 있습니다"],
  ["제가 이전 영상을 편집할 때 AI로 편집하고"]
]
```

## 분할 규칙

### 핵심 원칙
- **원문 단어를 그대로 유지** — 수정·재작성 금지. 타임스탬프 매핑에 필요하다.
- **출력 파트를 이어 붙이면 원문과 일치해야 한다** (공백 제외)

### 분할 기준 (우선순위 순)

1. **자연스러운 한국어 문법 경계**
   - 연결어미 뒤: `~하면`, `~하고`, `~해서`, `~하는데`, `~하다가`, `~하니까`, `~하지만`
   - 접속사 뒤: `그리고`, `하지만`, `그런데`, `그래서`, `그러면`, `따라서`, `즉`
   - 보조사 뒤: `은/는`, `이/가`, `을/를`, `에서`, `으로/로` (문장 흐름상 자연스럽다면)

2. **길이 기준** — 한 파트 **8~18자** 이내 (공백 포함)
   - 18자 이내라도 의미 단위가 완결되면 분할 가능
   - 한 세그먼트 전체가 15자 이하면 분할 없이 그대로 반환

3. **분할하지 말 것**
   - 명사구 중간 (`인공 / 지능`, `코드 / 편집기`)
   - 숫자+단위 (`3 / 분`, `50 / %`)
   - 조사와 앞 명사 (`편집 / 이`, `스킬 / 을`)
   - 영어 단어 중간 (`fast / er-whisper`)

### 예시

| 원문 | 분할 결과 |
|------|-----------|
| `클로드나 코덱스를 활용하면 컷편집과 자동자막을 자동으로 할 수 있습니다` | `["클로드나 코덱스를 활용하면", "컷편집과 자동자막을", "자동으로 할 수 있습니다"]` |
| `제가 이전 영상을 편집할 때 AI로 편집하고` | `["제가 이전 영상을 편집할 때", "AI로 편집하고"]` |
| `네 그래서` | `["네 그래서"]` (짧으면 분할 없이) |
| `CapCut에서 프로젝트를 만들고 영상을 타임라인에 올려주세요` | `["CapCut에서 프로젝트를 만들고", "영상을 타임라인에 올려주세요"]` |

## 작업 절차

### 1. 입력 파일 로드

```bash
cat /tmp/subtitle_input.json
```

### 2. 각 세그먼트 분할 결정

모든 세그먼트를 한 번에 읽고 **위에서 아래로 순서대로** 처리한다.

- 입력 세그먼트 수 = 출력 배열 수 (1:1 대응 필수)
- 분할이 필요 없는 세그먼트도 `["원문 그대로"]` 형태로 반드시 출력

### 3. 결과 저장

```python
import json

splits = [
    ["클로드나 코덱스를 활용하면", "컷편집과 자동자막을", "자동으로 할 수 있습니다"],
    ["제가 이전 영상을 편집할 때 AI로 편집하고"],
    # ... 입력 세그먼트 수만큼
]

with open("/tmp/subtitle_splits.json", "w", encoding="utf-8") as f:
    json.dump(splits, f, ensure_ascii=False, indent=2)
```

또는 Write 도구로 저장.

### 4. 완료 보고

```
✓ 자막 분할 완료
  - 처리 세그먼트: N개
  - 분할 발생: M개 세그먼트
  - 저장: /tmp/subtitle_splits.json
```

## 주의사항

- **입력과 출력의 세그먼트 수가 반드시 일치해야 한다** — `add_subtitles.py`가 인덱스로 매핑
- **단어를 수정하지 말 것** — 오타 교정은 `subtitle-verifier`의 역할
- 분할 경계에서 **앞뒤 공백이 달라도 무관** — 매핑 시 strip 처리함
